# ТЗ: HTTP API в 1С для зуботехнической лаборатории

**Кому:** 1С-программист  
**Цель:** Один раз настроить HTTP-сервис в 1С, чтобы внешнее веб-приложение (и мы с Claude) могли читать и обновлять данные через API — без повторных доработок 1С.

---

## 1. Что нужно сделать в 1С

Создать **HTTP-сервис** (раздел «Администрирование → HTTP-сервисы» в конфигураторе).

- Базовый URL сервиса: `/lab/hs/api/v1`
- Формат данных: **JSON** (Content-Type: application/json)
- Авторизация: **API-ключ** в заголовке каждого запроса:  
  `X-API-Key: <секретный_ключ>` — строку задаёт программист, передаёт мне.

Если запрос пришёл без ключа или с неверным — вернуть `401 Unauthorized`.

---

## 2. Список endpoint'ов (методов API)

### 2.1 Получить список техников

```
GET /lab/hs/api/v1/technicians
```

**Ответ:**
```json
{
  "technicians": [
    {
      "id": "Т00-000001",
      "name": "Иванов Иван Иванович",
      "city": "Сургут"
    },
    {
      "id": "Т00-000002",
      "name": "Петров Пётр Петрович",
      "city": "Москва"
    }
  ]
}
```

---

### 2.2 Получить заказы техника

```
GET /lab/hs/api/v1/orders?technician_id=Т00-000001
```

Если `technician_id` не передан — вернуть **все** заказы лаборатории (для дашборда руководителя).

Опциональные фильтры:
- `?status=в_работе` — только заказы с конкретным статусом
- `?from=2025-01-01&to=2025-12-31` — по диапазону дат сдачи

**Ответ:**
```json
{
  "orders": [
    {
      "id": "ЗЛ-000123",
      "patient_name": "Сидоров А.В.",
      "clinic_name": "Клиника Улыбка",
      "doctor_name": "Смирнов К.О.",
      "work_type": "Металлокерамика 3 ед.",
      "description": "Цвет A2, зубы 14-16, фото в папке заказа",
      "technician_id": "Т00-000001",
      "technician_name": "Иванов Иван Иванович",
      "status": "в_работе",
      "priority": "высокий",
      "created_at": "2025-06-01T10:00:00",
      "deadline": "2025-06-05T18:00:00",
      "appointment_at": "2025-06-06T14:00:00",
      "updated_at": "2025-06-02T09:30:00",
      "comments": [
        {
          "id": "1",
          "author": "Иванов Иван Иванович",
          "text": "Начал работу",
          "created_at": "2025-06-02T09:30:00"
        }
      ]
    }
  ],
  "total": 1
}
```

---

### 2.3 Получить один заказ

```
GET /lab/hs/api/v1/orders/ЗЛ-000123
```

**Ответ:** тот же объект заказа из п. 2.2, без обёртки `orders: []`.

---

### 2.4 Обновить статус заказа

```
PATCH /lab/hs/api/v1/orders/ЗЛ-000123/status
```

**Тело запроса:**
```json
{
  "status": "готово",
  "updated_by": "Т00-000001"
}
```

**Ответ при успехе:**
```json
{
  "success": true,
  "order_id": "ЗЛ-000123",
  "status": "готово"
}
```

---

### 2.5 Добавить комментарий к заказу

```
POST /lab/hs/api/v1/orders/ЗЛ-000123/comments
```

**Тело запроса:**
```json
{
  "author_id": "Т00-000001",
  "text": "Каркас готов, приступаю к керамике"
}
```

**Ответ:**
```json
{
  "success": true,
  "comment_id": "2",
  "created_at": "2025-06-03T11:00:00"
}
```

---


## 3. Допустимые значения статусов

| Код в API | Отображение |
|-----------|-------------|
| `новый` | Новый заказ |
| `в_работе` | В работе |
| `на_проверке` | На проверке |
| `готово` | Готово |
| `сдано` | Сдано врачу |

---

## 4. Допустимые значения приоритетов

| Код | Смысл |
|-----|-------|
| `высокий` | Дедлайн ≤ 1 суток |
| `средний` | Дедлайн 2-3 суток |
| `низкий` | Дедлайн > 3 суток |

Приоритет может считаться автоматически от `deadline`, если в 1С нет отдельного поля.

---

## 5. Коды ошибок

| HTTP-код | Когда возвращать |
|----------|-----------------|
| `200 OK` | Успешный запрос |
| `400 Bad Request` | Неверные параметры запроса |
| `401 Unauthorized` | Нет или неверный API-ключ |
| `404 Not Found` | Заказ или техник не найден |
| `500 Internal Server Error` | Ошибка на стороне 1С |

Формат ошибки:
```json
{
  "error": true,
  "code": 404,
  "message": "Заказ ЗЛ-000999 не найден"
}
```

---

## 6. Доступность

- Сервис должен быть доступен по внешнему IP (или через белый IP + порт публикации 1С).
- Либо через ngrok/обратный прокси — обсудим с программистом.
- Работа 24/7 (или хотя бы в рабочие часы 8:00–22:00).

---

## 7. Что передать мне после настройки

Программист должен прислать:
1. **Базовый URL** — например `http://185.10.xx.xx:8080/lab/hs/api/v1`
2. **API-ключ** — секретная строка для заголовка `X-API-Key`
3. **Реальные ID техников** из 1С (из endpoint `/technicians`)

---

## 8. Проверочный запрос (тест)

После настройки я проверю через:
```
curl -H "X-API-Key: <ключ>" http://<адрес>/lab/hs/api/v1/technicians
```

Если вернулся список — интеграция работает.
